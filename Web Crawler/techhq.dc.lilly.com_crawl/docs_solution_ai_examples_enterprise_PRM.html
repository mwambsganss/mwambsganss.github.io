An Agentic AI System for Research and Content Generation


Introductionâ€‹


Press Release Miner (PRM) is an advanced LLM-powered multi-agent system designed to extract insights from scientific source documents and news journals. Its primary purpose is to answer questions and generate briefing documents for key stakeholders using standard Eli Lilly language and style. The system employs a sophisticated agentic architecture that coordinates multiple specialized AI agents working together to conduct comprehensive research and produce high-quality content.


PRM addresses a critical need in corporate communications: conducting thorough research and generating media briefing documents for executives that ensure completeness, accuracy, and consistent messaging. By automating this process through an agentic approach, PRM significantly reduces the time and effort required while maintaining high standards of quality and consistency.

The PRM Processâ€‹


The Press Release Miner follows a structured six-step process to generate comprehensive briefing documents:


    

  1. 

    Develop a research plan: The system begins by analyzing the research topic and developing a strategic plan for gathering information. This includes identifying key factors and questions that need to be addressed.
    

    2. 

  2. 

    Search internal press releases: PRM leverages Retrieval Augmented Generation (RAG) to search through Eli Lilly's internal press releases, extracting relevant information and insights.
    

    3. 

  3. 

    Search web for news sources: The system extends its research to external sources, searching trusted news outlets and websites for additional context and information.
    

    4. 

  4. 

    Conduct follow-up research: Based on initial findings, PRM performs deeper, targeted research to fill knowledge gaps and answer specific questions that arise.
    

    5. 

  5. 

    Synthesize research results: The system integrates information from multiple sources, removing duplicates and organizing content into a coherent structure with proper citations.
    

    6. 

  6. 

    Generate briefing document: Finally, PRM produces a well-formatted briefing document that adheres to Eli Lilly's language and style guidelines.
    

    7. 



This entire process is orchestrated through approximately 50 LLM calls, distributed across various agents and tools within the system.


System Architectureâ€‹


PRM's architecture consists of four main components: UI interfaces, specialized agents, tools, and data sources. These components work together to create a seamless research and content generation pipeline.

Agentsâ€‹


PRM employs four specialized agents, each with distinct responsibilities:


    

  1. 

    Research Planner (Contextualization Agent): Responsible for understanding the research topic, breaking it down into manageable components, and developing a comprehensive research strategy. This agent provides the overall direction for the research process.
    

    2. 

  2. 

    Researcher (Research Agent/CoT/ToT): The primary information gathering agent that executes the research plan. It utilizes Chain-of-Thought (CoT) and Tree-of-Thought (ToT) reasoning to conduct thorough investigations across multiple sources.
    

    3. 

  3. 

    Synthesizer (Interview/Q&A Agent): Processes and integrates the collected information, identifying patterns, resolving contradictions, and organizing content into a coherent narrative. This agent can also generate follow-up questions to fill information gaps.
    

    4. 

  4. 

    Copyeditor (Formatting Agent): Ensures the final document adheres to Eli Lilly's style guidelines, maintains consistent formatting, and includes proper citations for all information sources.
    

    5. 



Toolsâ€‹


The agents leverage several tools to perform their tasks:


    

  1. 

    RAG (Retrieval Augmented Generation): Enables semantic search across internal documents, retrieving relevant information based on context rather than just keywords.

  2.

    3. 

  3. 

    YouTube: Extracts information from video content, particularly from news channels and official company presentations.
    

    4. 

  4. 

    Wikipedia: Provides background information and context for topics and entities mentioned in the research.

Agentic Patterns in PRMâ€‹


The Press Release Miner implements several sophisticated agentic patterns that enable effective coordination, parallel processing, and specialized task execution. These patterns represent the cutting edge of multi-agent system design and contribute significantly to PRM's effectiveness.


Multi-Agent Coordinationâ€‹


PRM's architecture demonstrates how multiple specialized agents can work together to accomplish complex tasks. The system implements a coordination mechanism that allows agents to:


    

  1. 

    Share context: Agents pass information and context to each other, ensuring continuity throughout the research process.
    

    2. 

  2. 

    Sequential processing: The Research Planner sets the direction, followed by the Researcher gathering information, the Synthesizer integrating it, and finally the Copyeditor formatting the output.
    

    3. 

  3. 

    Feedback loops: Agents can request additional information or clarification from other agents when needed, creating a dynamic research process.
    

    4. 



This coordination pattern enables each agent to focus on its specialized task while contributing to the overall goal of producing a comprehensive briefing document.


Tool Usage Patternsâ€‹


PRM implements three distinct tool usage patterns across its agents:


    

  1. Tool Calling Pattern: The system uses a structured approach for agents to invoke tools, with clear input/output specifications. This pattern is implemented in tool_calling_tool.py, which creates a runnable agent that can use multiple tools based on a system template and human instructions.
    2. 



function tool_calling_runnable(tool_runtime):
    Initialize system template from tool_runtime
    Copy and initialize tools from tool_runtime
    For each tool:
        Add tool name and description to system template
    Create prompt template with system template, human template, and agent scratchpad
    Create tool-calling agent with LLM, tools, and prompt
    Return agent executor with configured agent and tools


    

  1. ReAct Pattern (Reasoning and Acting): PRM implements the ReAct pattern in react_tool.py, allowing agents to alternate between reasoning about their next steps and taking actions. This pattern enhances the agent's ability to plan and execute complex research tasks.
    2. 



function react_agent_runnable(tool_runtime):
    Copy and initialize tools from tool_runtime
    Create prompt from react template
    Create ReAct agent with LLM, tools, and prompt
    Return agent executor with configured agent and tools


    

  1. Self-Ask Pattern: Implemented in self_ask_tool.py, this pattern enables agents to break down complex questions into simpler sub-questions, answer them sequentially, and then synthesize a final answer. This is particularly useful for complex research topics that require multi-step reasoning.
    2. 



function self_ask_runnable(tool_runtime):
    Copy tools from tool_runtime
    Select first tool and rename it to "Intermediate Answer"
    Create prompt from self-ask template
    Create self-ask agent with LLM, tool, and prompt
    Return agent executor with configured agent and tool


Parallel Processingâ€‹


One of PRM's most powerful features is its ability to conduct research on multiple topics or factors simultaneously. This is implemented through the parallelization_tool.py, which enables:


    

  1. 

    Concurrent tool execution: Multiple tools can be executed in parallel across different research topics.
    

    2. 

  2. 

    Asynchronous processing: The system uses Python's asyncio to manage concurrent tasks efficiently.
    

    3. 

  3. 

    Result aggregation: Results from parallel processes are collected and integrated into a coherent output.
    

    4. 



async function async_parallel_tool_call(topics, tool_runtime, id_task, target):
    Create queue for communication
    Create empty tasks list
    For each topic:
        Create async task for processing topic and add to tasks list
    
    While true:
        Get item from queue
        Yield item to caller
        Mark task as done
        
        If all tasks are done and queue is empty:
            Break loop
    
    Await all tasks to complete


This parallel processing capability significantly reduces the time required to conduct comprehensive research across multiple topics or factors.


Agent Specialization and Collaborationâ€‹


PRM demonstrates how specialized agents can collaborate effectively to produce high-quality outputs. Each agent has a specific role:


    

  1. Research Pipeline Agent: Orchestrates the overall research process, managing the flow of information between other agents and tools.

        Select appropriate agent type
        Process research results in parallel
        Format and deduplicate results
        Yield final output


    

  1. Formatting Agent: Specializes in ensuring consistent formatting and removing duplicates across research outputs.

Technical Implementationâ€‹


LLM Integrationâ€‹


PRM integrates with Azure OpenAI's GPT-4o model to power its agents. The system makes approximately 50 LLM calls throughout the research and content generation process, distributed across various agents and tasks. The LLM integration is managed through a custom model class that handles authentication and API interactions with Azure OpenAI services and AWS resources.


Technologies Usedâ€‹


PRM leverages several key technologies:


    

  1. 

    LangChain: Used extensively for agent creation, tool integration, and prompt management. LangChain provides the foundation for the agentic patterns implemented in PRM.
    

    2. 

  2. 

    FastAPI: Powers the backend API, providing WebSocket endpoints for real-time communication during the research process.
    

    3. 

  3. 

    FAISS: Implements the vector store for efficient semantic search across internal documents.
    

    4. 

  4. 

    Azure OpenAI: Provides the LLM capabilities that power the agents' reasoning and content generation.
    

    5. 

  5. 

    Bing Search API: Enables web search functionality for gathering information from external sources.

Prompt Engineeringâ€‹


Prompt engineering is a critical aspect of PRM's implementation. The system uses carefully crafted prompts to guide each agent's behavior and ensure high-quality outputs. For example:


    

  1. Research Agent Prompt: Guides the research process with specific instructions for gathering comprehensive information:
    2. 



# The research agent prompt instructs the agent to gather detailed information with proper citations, format responses with chronological ordering and bulleted lists, and ensure all facts are properly attributed to their sources.


    

  1. Formatting Agent Prompt: Ensures consistent formatting and structure across research outputs:
    2. 



# The formatting agent prompt guides the agent to combine research outputs from multiple sources into a cohesive document, maintain proper citations for all facts and figures, and structure the content with appropriate headings and subheadings.


    

  1. De-duplication Prompt: Removes redundant information across research outputs:
    2. 



# The de-duplication prompt instructs the agent to compare reference and candidate texts, remove duplicate facts, quotes, and figures from the candidate text, and ensure the final output has no redundant information while maintaining coherent flow.


These prompts are critical for guiding the agents' behavior and ensuring they produce outputs that meet Eli Lilly's standards for quality and consistency.


Quality Control Mechanismsâ€‹


PRM implements several quality control mechanisms to ensure the accuracy and consistency of its outputs:


    

  1. Uniform Formatting: The FormatResearchAgent ensures consistent formatting across all research outputs:
    2. 



async function async_format_section(research_output, format_section_prompt):
    Create async queue for communication
    Find reference topic with longest text
    Create formatting instruction template
    Create formatting chain with LLM
    
    For each topic and text:
        If not reference topic:
            Create task to format candidate text
        Else:
            Add to output without formatting
    
    Process queue until all tasks complete
    Return formatted research outputs


    

  1. De-duplication: Removes redundant information across research outputs:
    2. 



async function async_remove_duplicates(research_output, remove_duplicates_prompt):
    Create async queue for communication
    Create deduplication instruction template
    Create deduplication chain with LLM
    Track processed pairs to avoid duplicate work
    
    For each reference topic/text pair:
        For each candidate topic/text pair:
            If not same topic and not already processed:
                Create task to deduplicate candidate text
                Mark pair as processed
            Else:
                Add to output without deduplication
    
    Process queue until all tasks complete
    Return deduplicated research outputs


    

  1. Citation Requirements: Ensures all information is properly cited with source details:
    2. 



The system enforces strict citation requirements, ensuring that every fact, figure, date, and quote is properly attributed to its source with a standardized citation format that includes the source name, publication date, headline, and full web address.


These quality control mechanisms ensure that PRM produces outputs that are accurate, well-structured, and properly cited.
