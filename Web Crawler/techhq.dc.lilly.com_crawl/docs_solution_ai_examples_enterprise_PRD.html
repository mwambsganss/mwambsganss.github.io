An Agentic AI System for Press Release Generation


Product Overviewâ€‹


The Press Release Drafter (PRD) is an agentic AI product designed for Lilly's corporate communications team to automate
the generation of preliminary press releases for clinical trial results. This advanced system leverages multi-agent
reasoning and natural language processing to create accurate, compliant, and professionally styled press releases that
align with Lilly's tone and communication standards.


Problem Statementâ€‹


Creating press releases for clinical trial results is a time-consuming process that requires:


    

  • Extracting and synthesizing complex clinical data from various sources
    • 

  • Ensuring medical and regulatory compliance
    • 

  • Maintaining consistent messaging and brand voice
    • 

  • Coordinating input from multiple stakeholders
    • 

  • Meeting tight deadlines for market-sensitive information
    • 



This process currently demands significant time from skilled communications professionals, medical reviewers, and
regulatory experts, often leading to bottlenecks in the release pipeline.


Target Usersâ€‹


    

  • Corporate Communications team members
    • 

  • Medical Affairs professionals
    • 

  • Regulatory Affairs specialists
    • 

  • Clinical Trial teams
    • 

  • Public Relations managers
    • 



Product Goalsâ€‹


    

  1. Reduce the time required to create initial press release drafts by 70%
    2. 

  2. Ensure 100% compliance with regulatory requirements and Lilly's communication guidelines
    3. 

  3. Improve consistency in messaging across all clinical trial press releases
    4. 

  4. Enable communications team to focus on strategic messaging rather than initial drafting
    5. 

  5. Accelerate the review cycle by providing higher quality first drafts
    6. 



Key Features and Requirementsâ€‹


1. Multi-Agent Reasoning Systemâ€‹


    

  • Specialized agents for data extraction, medical accuracy, regulatory compliance, and content generation
(Supervisor-Worker Agent Pattern)
    • 

  • Collaborative reasoning between agents to resolve conflicts and ensure coherence
(Peer Network Collaboration Pattern)
    • 

  • Transparent decision-making process with justifications for content choices
    • 



2. Advanced Insights Extractionâ€‹


    

  • Support for multiple data source formats:

      

    • Text documents (TXT, PDF)
      • 

    • Presentations (PPTX)
      • 

    • Spreadsheets (XLSX)
      • 

    • Images and charts
      • 

    • Word documents (DOCX)
      • 

    

    • 

  • Automated extraction of key clinical trial data points
(ReAct Pattern):

      

    • Primary and secondary endpoints
      • 

    • Safety profiles
      • 

    • Statistical significance
      • 

    • Patient demographics
      • 

    • Study design details
      • 

    

    • 



3. MLR Guardrails for Complianceâ€‹


    

  • Built-in Medical, Legal, and Regulatory (MLR) checks
(Self Critique Pattern)
    • 

  • Automatic flagging of potential compliance issues
    • 

  • Citation and reference management
    • 

  • Verification of claims against source data
    • 

  • Adverse event reporting compliance
    • 



4. Dynamic Content Adjustmentâ€‹


    

  • Configurable tone settings (formal, conversational, technical)
    • 

  • Adjustable draft length (brief announcement, detailed release, comprehensive background)
    • 

  • Audience-specific language calibration
    • 

  • Customizable emphasis on different aspects of trial results
    • 



5. Rich Authoring Experienceâ€‹


    

  • Interactive editing interface
    • 

  • Real-time suggestions and alternatives
    • 

  • Collaborative review capabilities
    • 

  • Version control and change tracking
    • 

  • Comment and feedback integration
    •

Technical Requirementsâ€‹


Integration Requirementsâ€‹


    

  • Secure connection to clinical trial databases
    • 

  • Integration with document management systems
    • 

  • API access to regulatory guidelines database
    • 

  • Connection to Lilly's brand voice guidelines
    • 



Security and Complianceâ€‹


    

  • End-to-end encryption for all data
    • 

  • Role-based access controls
    • 

  • Audit trail for all content generation and edits
    • 

  • Compliance with data privacy regulations
    • 

  • Secure handling of pre-public material information
    • 



Performance Requirementsâ€‹


    

  • Initial draft generation in under 5 minutes
    • 

  • Support for concurrent users across global teams
    • 

  • 99.9% uptime during business hours
    •

Conclusionâ€‹

efficiency and effectiveness of Lilly's external communications.
