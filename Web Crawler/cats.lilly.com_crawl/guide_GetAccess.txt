Get Access
To start deploying on the CATS Platform you will need to be added to a few different groups. We have convienently automated this proccess via the Developer
FRONTDOOR
! All you need to do is click the buttons to be automatically added to each group.
1. Github Group
​
Group Name
: LRL_light_k8s_infra_write
Location
:
Github
Description
: This is a Github group that provides users access to the repositories where deployment resources are defined. Once you are in this group you can see and work with the github repositories associated with the CATS Platform.
Request Access
- Follow the steps below to gain access
Have a Github account that is a member of the Eli Lilly Github Organization. Follow
these steps
if that is not completed.
Navigate to the
FRONTDOOR
and click the desginated button on the CATS Page to get added to the group.
If you have a Github Account you will be automatically added to the group. If you do not have a Github account the automation will fail, send you an email with instructions on how to set up a Github account and have your account added to the Eli Lilly Github Organization. You will be instructed to try again once you have successfully set up your Github account.
2. Dashboards Group
​
Group Name
: lrl_cats_access_non_rids
Location
:
AD Group Manager
Description
: This is an AD group that provides users access to various dashboards related to the CATS Platform.
Request Access
- Navigate to the
FRONTDOOR
and click the desginated button on the CATS Page to get added to the group.
3. AWS Console and Cluster Access Group (CA Account)
​
Group Name
: aws_light_devs
Location
:
AD Group Manager
Description
: This is an AD group that requires you to have a CA account in order to gain access. Once you are a member of this group it will allow you to access the AWS Accounts where CATS is deployed and use many features in the AWS Console.
Request Access
- Follow the steps below to gain access
Get a
CA Account
Navigate to the
FRONTDOOR
and click the desginated button on the CATS Page to get added to the group.
If you have a CA Account you will be automatically added to the group. If you do not have a CA account the automation will fail, send you an email with instructions on how to get a CA account, and tell you to try again.
4. Mailing List
​
Group Name
: cats_mailing_list
Location
:
AD Group Manager
Description
: When a member of this group the user will recieve communications related to the CATS Platform. Communications include notifications of downtimes, platform changes, release notes, etc.
Request Access
- Navigate to the
FRONTDOOR
and click the desginated button on the CATS Page to get added to the group.
Sign In with CA Account
​
Many users struggle to sign in with their CA Account for the first time. The following workflow will help users sign in to the AWS Console but the same process is used to sign into any other service with the CA Account.
have a CA account. You CA account will be structured like this:
C233778-CA@llynet.com
. You should replace your number and the ending will always be
-CA@llynet.com
.
When opening the AWS console it is important that you do not navigate to the portal when you are already logged in with your normal profile. This happens to many users. It is important that you open a private browser where you are not logged in with any accounts and then navigate to the
AWS Portal
. When you get to the portal it will ask you to sign in. Be sure to sign in with your CA account. something like
C233778-CA@llynet.com
.
The system will then ask you for your password. To get your password you need to go to a different browser and log into
CyberArk
with your normal account. Once you are logged in and have your password you use it to log into the AWS Portal. To find your password select which account you want to see the password for then click the show button. Copy your password and then paste it into the prompt in the private window to proceed.
If you are in the aws_light_devs group you should see a landing page that looks like this:
select which AWS account you want to login with and then select the role "aws_light_devs"
You should be in the console and ready to go! To understand which AWS Accounts map to the various CATS Deployment Repos please take a look at this table:
